# Attorney Assistant Source-of-Truth Matrix

> Package 1 audit baseline. The final runtime status and remaining user-facing limitations are recorded in `FINAL_CAPABILITY_CONTRACT.md`. This audit is retained as historical evidence and must not be read as the current implementation status.

Audit date: July 22, 2026

Scope: authenticated, approved attorney support drawer only. This inventory describes what the product currently does, what evidence the assistant currently has, and what Package 2 must add. It does not authorize record-changing assistant actions.

Status meanings:

- `Ready`: current evidence path is sufficient for the stated read-only answer.
- `Partial`: some states are answerable, but the evidence contract is incomplete or unsafe for the full question family.
- `Gap`: the product contains the workflow, but the attorney assistant lacks an authoritative tool or response contract.
- `Boundary`: the assistant must refuse or redirect without implying it performed work.

## Required Source Precedence

1. Authenticated ownership and role enforcement.
2. Shared executable workflow policy used by the enforcing backend route.
3. Live scoped account, matter, processor, or storage evidence.
4. Stored immutable financial or lifecycle snapshot.
5. Approved knowledge content that does not conflict with levels 1–4.
6. Truthful limitation or one focused clarification.

Conversation history, page copy, previous assistant messages, and model memory are never authoritative evidence.

## Current Coverage Conclusion

The existing 18-capability manifest is a useful routing baseline but is not a complete attorney product contract. This audit defines 32 question families. Some narrow substates can already be answered from verified data, but no full question family is classified `Ready` across all material states. Every family is therefore explicitly partial, a gap, unsafe/policy-blocked, or a boundary until the material issues above are resolved and verified in later packages.
